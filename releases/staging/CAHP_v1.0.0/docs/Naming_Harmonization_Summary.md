## Key Points:
- All documentation files are now named with the `.helix.md` suffix.
- Files contain only lowercase letters, underscores, and appropriate version identifiers.
- Media files (images, videos) and code files are excluded from renaming to preserve internal links and references.
- Internal links and references are verified for correctness.

## Responsible Parties:
- All team members are expected to follow the naming conventions in new content.
- Existing files should be reviewed regularly for compliance.
- Documentation references and internal links must be updated to match filename changes.
